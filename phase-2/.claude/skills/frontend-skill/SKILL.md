# Frontend Skill

## Instructions

1. **Page Structure**
   - Plan page layout with header, main content, and footer
   - Use semantic HTML elements (`<header>`, `<main>`, `<section>`, `<footer>`)
   - Ensure responsive design for mobile, tablet, and desktop

2. **Components**
   - Break UI into reusable components (buttons, cards, modals, forms)
   - Pass data via props for dynamic rendering
   - Keep components modular and isolated
   - Use component libraries (optional) for speed

3. **Styling**
   - Use CSS, SCSS, or utility frameworks like Tailwind CSS
   - Apply consistent colors, spacing, and typography
   - Use flexbox and grid for layout alignment
   - Handle hover, focus, and active states for interactivity

4. **Interactivity**
   - Add animations and transitions for smooth UI experience
   - Handle user events like clicks, inputs, and form submissions
   - Update component state dynamically (React, Vue, etc.)
   - Use conditional rendering for dynamic content

## Best Practices
- Keep components small and reusable
- Follow mobile-first design principles
- Use semantic HTML for accessibility
- Maintain consistent naming conventions for classes and components
- Optimize images and assets for performance
